    Crear Pull Request

    Paso Captura de pantalla
    Click the Pull Request icon on the sidebar, then from the Pull Request page, click the green New pull request button.
    Select the branch you made, readme-edits, to compare with master (the original).
    Look over your changes in the diffs on the Compare page, make sure they’re what you want to submit.
    When you’re satisfied that these are the changes you want to submit, click the big green Create Pull Request button.
    Give your pull request a title and since it relates directly to an open issue, include “fixes #” and the issue number in the title. Write a brief description of your changes.

    Organización Directorios Subdirección

    Con el ánimo de estandarizar el código y hacer mucha más fácil la re-utilización del mismo en diferentes proyectos que se ejecutan en la Subdirección de Estadística del ICFES, se recomienda a los usuarios de la subdirección seguir la siguiente estructura de carpetas, de acuerdo al tipo de proyecto en el que estén trabajando. Siguiendo esta estructura de almacenamiento será más sencillo hacer búsquedas por diferentes usuarios, además de facilitar la replicabilidad, auditoría y entendimiento del trabajo realizado (RAE).

    Proyectos

    Es la estructura general de cualquier tipo proyecto o análisis desarrollado por los integrantes de la subdirección:

    • Doc: Todo tipo de documentación sobre el proyecto o sobre el proceso general de esa tarea. También debe estar incluidos los reporte de resultados de los proyectos (Ejemplo análisis de ítems).
    • Input: Las referencias a conjuntos de datos utilizados , es decir , enlaces y descripción general
    • Output: Los resultados experimentales, los archivos con tablas , figuras, etc, datos temporales.
    • Src: El código fuente separados por lenguaje de programación.
    • Sweave: Se deben organizar los códigos que se utilizan en la maquetación de reportes, ya sea con la herramienta Sweave de R o con código escrito en Markdown (ver Leguaje de marcadores).
    • Perl: Como ya se había mencionada el código fuente debe estar separado por lenguaje de programación, en este ejemplo el proyecto utiliza códigos programados en Perl, es por esto que se debe incluir una nueva carpeta.

    Artículos

    En la sección Documentos debe almacenar la producción los artículos de la Subdirección en el código fuente (.tex) y la versión en .pdf.

    • Imágenes: Cada imagen que se utiliza en el documento debería estar allí.
    • Scripts: Scripts y plantillas utilizadas para fabricar el articulo.
    • Los datos: Datos adicionales o aquellos que se quieran compartir en el repositorio, son los resultados del articulo.

    Equipos de Trabajo

    tomado de http://yakiloo.com/getting-started-github/

    Las personas que son propietarios o tienen el rol de administraodr, pueden el acceso repositorio con los equipos . Cada equipo puede tener diferentes permisos de acceso repositorio.

    Hay tres tipos de permisos de repositorios disponibles para las personas o equipos que colaboran, en los repositorios que pertenecen a una organización:

    *Permisos de Lectura *Permisos de Escritura *Permisos de Administración

    Además, miembros de la organización con permisos de propietario tienen amplios permisos en todos los repositorios de una organización. Para obtener más información, consulte [Los niveles de permisos para una organización][https://help.github.com/articles/repository-permission-levels-for-an-organization/].
