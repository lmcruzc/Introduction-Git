Paso Captura de pantalla
Click the Pull Request icon on the sidebar, then from the Pull Request page, click the green New pull request button.
Select the branch you made, readme-edits, to compare with master (the original).
Look over your changes in the diffs on the Compare page, make sure they’re what you want to submit.
When you’re satisfied that these are the changes you want to submit, click the big green Create Pull Request button.
Give your pull request a title and since it relates directly to an open issue, include “fixes #” and the issue number in the title. Write a brief description of your changes.

  2. Dentro de la carpeta "Ejercicio_Individual", crear una carpeta con el siguiente formato "NOMBRE_APELLIDO" y agregar un archivo de preferencia (Script, Imagen, Pdf, etc..).

  4. Hacer un reporte de errores, a uno de los archivos de otro Usuario.

  6. Hacer Merge con el Branch Principal.

Equipos de Trabajo

tomado de http://yakiloo.com/getting-started-github/

Las personas que son propietarios o tienen el rol de administrador, pueden dar el acceso a los repositorio y administrar los equipos. Un equipo puede tener diferentes permisos de acceso al repositorio.

Existen tres tipos de niveles de niveles de acceso, que pueden ser otorgados a usuario o equipos que pertenecen a una organización.

  • Permisos de Lectura
  • Permisos de Escritura
  • Permisos de Administración

Además, miembros de la organización con permisos de propietario tienen amplios permisos en todos los repositorios de una organización. Para obtener más información, consulte Niveles de Permisos
